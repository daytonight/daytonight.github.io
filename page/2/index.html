本次建站心得感悟

1.深刻意识到信息检索的重要性，必应是最佳答案最多的搜索引擎；

2.一开始是没有建站环境的概念的，后来了解到很多人用jekyll就试着弄了以下，结果是搭建环境太繁琐，有的指令之前管用，过一会儿就莫名其妙的不行，后来就果断寻找其他方式搭建环境；

3.hexo和jekyll比起来少了很多的输指令的步骤，在安装上就简单直接很多，并且模板也挺多，hexo的发现对于我这种纯小白用户是可以欣喜若狂的，虽然目前只停留在简单的编辑文章上，还有很多要学；

4.期间参考了部分网页制作的书，大概了解了HTML和css语法，再后来就了解到另一件小白利器：markdown大致看了下markdown语法，学会了基础的写法

5.神编辑器notepad++也帮到了不少，可打开和编辑各种格式文件；

总的来说，这次建站经历还是挺有收获的，各种知识都是息息相关，要在茫茫多的信息中尽快找到自己需要的是一件不容易的事情，今后要多多锻炼这方面的能力。

windows+hexo创建github博客

hexo环境下创建github博客

如何搭建hexo环境

下载git

下载对应系统的node.js安装包并安装

手动创建工作目录hexo运行如下代码

hexo init
hexo server

此时本地服务器已能看到静态网页

复制好看的主题

参见如何搭建一个独立博客

复制完后需要更改_config.yml文件，最好的方法是直接复制过来，然后执行

cd jacman
git pull

将本地静态网页写到github上

编辑—config.yml

deploy:
  type: github
  repository: your URL
  branch: master

执行下列指令完成部署

hexo g
hexo d

以上步骤执行完后会要求输入用户名和密码，完成输入后进入：
username.github.io即能看到新生成的博客页面

新用户可重新设置ssh

此后可执行以下代码以向博客中添加文章

hexo new
hexo g
hexo d

常用指令

  • hexo init:用git bash打开执行后自动生成建站所需所有文件

  • npm install：与上一条指令联合使用

  • npm instll hexo-deployer-git —save:一般hexo deploy出错的时候要执行这条指令，在这里提前安装好

  • hexo g:生成文章
  • hexo s:启动本地服务，预览前面提到的静态网页
  • hexo d:部署
