python项目四-新闻聚合

By daytonight

发表于 2015-12-06

基本原理:编写一个信息收集代理程序，通过一个USENET专用服务器，从一个指定的USENET新闻组收集信息并保存到指定的目的文件夹中。

USENET是一个世界性的新闻组网络系统，是一个巨大无比的网上讨论组，下面对USENET作几点说明：

  • 运行机制：
    USENET的运行机制对于用户来说，分为以下三步。 　
    1. 网络服务提供商（ISP）在一个网络中，设定一台服务器作为USENET专用服务器，再将它的网址告诉用户。
    2. 用户想要发言的时候，就向这个网址发送帖子（post），这与发送Email很相似，但是两者格式不一样，在USENET上发言必须使用专用的客户端。不过，现在大多数的Email客户端都带有新闻组功能，最常见的Outlook Express的设置可以参考网上的说明。
    3. 查看其他人的发言时，就必须从服务器上下载其他人的帖子。下载完成后，如果想回复某人的帖子，就再重复第二步。
      可以看到，这个过程同邮件列表的运行几乎一模一样，不同之处在于，USENET服务器每天会同其他USENET服务器交换帖子。这就是说，全世界所有的USENET服务器最终都可以互相交换帖子，保持内容的同步。所以理论上，不管你的帖子是发到哪一台服务器上，最终全世界的人们都会看到，并且会从世界各地给你回复。因此，USENET就有一个其他交流机制所没有的优点，即这是一个真正的全世界参与的讨论组。
  • 内容结构：
     1. 由于USENET中的讨论内容无所不包，所以必须根据主题分类。每一个主题就是一个”频道”，对这个主题感兴趣的用户就订阅这个频道。
      2. USENET中的主题分类采用等级制（hierarchies），在形式上同域名很相似，即”一级主题.二级主题.三级主题….”，中间以小数点分隔。（comp.lang.python.announce）
     一级主题有九个，分别为：
    comp.: 与计算机相关的讨论。
    misc.    : 各种不属于其他分类的主题。
    news.: 对USENET本身的讨论
    rec.    : 休闲和娱乐
    sci.:与科学相关的讨论。
    soc.    : 与社会相关的讨论。
    talk.: 各种争议性话题的讨论。
    humanities.    : 艺术、文学、哲学方面的讨论。
    alt.*: 自由讨论区。

from nntplib import NNTP
from time import strftime,time,localtime
from email import message_from_string
from urllib import urlopen
import textwrap
import re

day = 24*60*60

def wrap(string,max=70):
    '''

    '''
    return '\n'.join(textwrap.wrap(string)) + '\n'

class NewsAgent:
    '''
    '''
    def __init__(self):
            self.sources = []
            self.destinations = []

    def addSource(self,source):
            self.sources.append(source)

    def addDestination(self,dest):
            self.destinations.append(dest)

    def distribute(self):

            items = []
            for source in self.sources:
                    items.extend(source.getItems())
            for dest in self.destinations:
                    dest.receiveItems(items)

class NewsItem:
    def __init__(self,title,body):
            self.title = title
            self.body = body

class NNTPSource:
    def __init__(self,servername,group,window):
            self.servername = servername
            self.group = group
            self.window = window

    def getItems(self):
            start = localtime(time() - self.window*day)
            date = strftime('%y%m%d',start)
            hour = strftime('%H%M%S',start)

            server = NNTP(self.servername)

            ids = server.newnews(self.group,date,hour)[1]

            for id in ids:
                    lines = server.article(id)[3]
                    message = message_from_string('\n'.join(lines))

                    title = message['subject']
                    body = message.get_payload()
                    if message.is_multipart():
                            body = body[0]

                    yield NewsItem(title,body)

            server.quit()

class SimpleWebSource:

    def __init__(self,url,titlePattern,bodyPattern):
            self.url = url
            self.titlePattern = re.compile(titlePattern)
            self.bodyPattern = re.compile(bodyPattern)

    def getItems(self):
            text = urlopen(self.url).read()
            titles = self.titlePattern.findall(text)
            bodies = self.bodyPattern.findall(text)
            for title.body in zip(titles,bodies):
                    yield NewsItem(title,wrap(body))

class PlainDestination:

    def receiveItems(self,items):

                    print item.body

class HTMLDestination:

            print >> out, '<ul>'

            for item in items:
                    id += 1

                    id += 1

def runDefaultSetup():

    agent = NewsAgent()

    bbc_url = 'http://news.bbc.co.uk/text_only.stm'
    bbc_title = r'(?s)a href="[^"]*">\s*<b>\s*(.*?)\s*</b>'
    bbc_body = r'(?s)</a>\s*<br/>\s*(.*?)\s*<'

    clpa_server = 'news2.neva.ru'
    clpa_group = 'alt.sex.telephone'

    agent.addSource(clpa)

from xml.sax.handler import ContentHandler
from xml.sax import parse
import os

class Dispatcher:
        def dispatch(self, prefix, name, attrs=None):
                mname = prefix + name.capitalize()
                dname = 'default' + prefix.capitalize()
                method = getattr(self, mname, None)
                if callable(method): args = ()
                else:
                        method = getattr(self, dname, None)
                        args = name,
                if prefix == 'start': args += attrs,
                if callable(method): method(*args)

        def startElement(self, name, attrs):
                self.dispatch('start', name, attrs)

        def endElement(self, name):
                self.dispatch('end', name)

class WebsiteConstructor(Dispatcher, ContentHandler):
        passthrough = False

        def __init__(self, directory):
                self.directory = [directory]
                self.ensureDirectory()

        def ensureDirectory(self):
                path = os.path.join(*self.directory)
                print path
                print '----'
                if not os.path.isdir(path): os.makedirs(path)

        def characters(self, chars):
                if self.passthrough: self.out.write(chars)

        def defaultStart(self, name, attrs):
                if self.passthrough:
                        self.out.write('<' + name)
                        for key, val in attrs.items():
                                self.out.write(' %s="%s"' %(key, val))
                        self.out.write('>')
        def defaultEnd(self, name):
                if self.passthrough:
                        self.out.write('</%s>' % name)

        def startDirectory(self, attrs):
                self.directory.append(attrs['name'])
                self.ensureDirectory()

        def endDirectory(self):
                print 'endDirectory'
                self.directory.pop()

        def startPage(self, attrs):
                print 'startPage'
                filename = os.path.join(*self.directory + [attrs['name']+'.html'])
                self.out = open(filename, 'w')
                self.writeHeader(attrs['title'])
                self.passthrough = True

        def endPage(self):
                print 'endPage'
                self.passthrough = False
                self.writeFooter()
                self.out.close()

        def writeHeader(self, title):
                self.out.write('<html>\n <head>\n   <title>')
                self.out.write(title)
                self.out.write('</title>\n </head>\n  <body>\n')

        def writeFooter(self):
                self.out.write('\n </body>\n</html>\n')

Python切片方法：

       number=[1,2,3,4,5,6,7,8,9]

  1. 正向切片(Step>0)：
    number[indexL,indexR,Step]
    左序检索：indexL=（0,8）、indexR=（1,9）
    右序检索：indexL=（-1,9）、indexR=(0，-9)
    例：
    number[1,5]=[2,3,4,5]
    number[1:]=[2,3,4,5,6,7,8,9]
    number[:]=[1,2,3,4,5,6,7,8,9]
    number[0,-2]=[1,2,3,4,5,6,7]
    number[-3,-1]=7,8
    若左边的索引（indexL）比右边的索引晚出现在序列中（从左往右看），结果将是空序列：
    number[-3:0]=[]
    步长(Step)>0(不能=0)，将按步长遍历:
    number[0:9:2]=1,3,5,7,9

  2. 反向切片（Step<0）:
    左序检索：indexL=(0，8)、indexR=（-1,8）
    右序检索：indexL=（-1,8）、indexR=（-2，-10）

    number[8:3:-1]=9,8,7,6,5,
    number[8:-3:-1]=9,8
    number[:5:-1]=9,8,7
    nunmber[5::-1]=6,5,4,3,2,1
    number[5:1:-1]=6,5,4,3
    若左边的索引（indexL）比右边的索引（indexR）晚出现在序列中（从右往左边看），结果将是空序列
    ：number[2:-2]=[]

#coding:utf-8
class Rule():
    '''
    所有规则的基类
    '''
    def action(self,block,handler):
       handler.start(self.type)
        handler.feed(block)
        handler.end(self.type)
        return True

class HeadingRule(Rule):
    '''
    标题占以行，最多70个字符，并且不以冒号结尾。
    '''
    type='heading'
    def condition(self,block):
        return not '\n' in block and len(block)<=70 and not block[-1]==':'

class TitleRule(HeadingRule):
    '''
    题目是文档的第一个块，但前提是他是大标题。
      '''
    type ='title'
    first =True

    def condition(self,block):
        if not self.frist:return False
        self.first=False
        return HeadingRule.condition(self,block)

class ListItemRule(Rule):
    '''列表项是以连字符开始的段落。作为格式化的一部分，要一处连字符
      '''
    type='listitem'
    def condition(self,block):
        return block[0]=='-'
    def action(self,block,handler):
        handler.start(self.type)
        handler.feed(block[1:].strip())#去除连字符 
        handler.end(self.type)
        return True

class ListRule(ListItemRule):
    '''
    列表从不是列表项的块和随后的列表项之间。在最后一个连续列表项之后结束
    '''
    type='list'
    inside=False
    def action(self,block,handler):
        if not self.inside and ListItemRule.condition(self,block):
            handler.start(self.type)
            self.inside=True
        elif self.inside and not ListItemRule.condition(self,block):
            handler.end(self.type)
            self.inside=False
        return False

class ParagraphRule(Rule):
    '''
    段落只是其他规则并没有覆盖到的块
    '''
    type='parapraph'
    def condition(self,block):
        return True

计算机发展历史简述

By daytonight

发表于 2015-07-30

 计算机是一种能迅速而高效的自动完成信息处理的电子设备，它能按照程序对信息进行加工、处理、存储。计算机自诞生至今已有约七十年的历史，应用于社会上各个领域，对人类科学文明的发展产生了革命性的的影响，其软硬件更新也发生了翻天覆地的变化，根据制造电子计算机采用的物理器件的不同,可以将计算机的发展过程分为如下四个阶段:

  1. 第一代计算机（1946-1957年）
     第一代计算机使用的逻辑器件是电子管，电子管会大量发热，且体积较大，同时采用磁鼓作为主存储器，磁带作为辅助存储设备。这一时期的计算机的特点是体积大、运算速度低、存储量小、可靠性低，几乎没有软件配置。世界上第一台数字计算机ENIAC，于1946年被美国宾夕法尼亚大学研制出来，代表了计算机发展史上的里程碑。

  2. 第二代计算机（1959-1965年）
      第二代计算机使用的逻辑器件主要是晶体管，采用磁芯作为存储器，磁盘作为辅助存储设备，同时引进了编制寄存器和浮点运算硬件，利用I/O处理提高了输入输出的效率。这不仅使计算机的体积缩小了许多，同时增加了机器的稳定性并提高了运算速度，减小了功耗，降低了价格。

  3. 第三代计算机（1965-1971年）
     第三代计算机使用的逻辑器件是集成电路，较之第二代的印刷电路体积更小，更廉价，更快速和可靠。晶体管也应用到构造存储器上做成半导体存储器，由于晶体管存储器断电后信息会丢失，辅助存储设备仍是必需的。同时出现了带有键盘和屏幕的I/O设备，计算机开始走向标准化、模块化和系列化。

  4. 第四代计算机（1971年至今）
     第四代计算机的特征是使用了大规模和超大规模集成电路，使计算沿着两个方向发展。一方面利用大规模集成电路生产多种逻辑芯片，组装大型、巨型计算机，使运算速度向每秒万亿次及更高发展，存储容量向百兆、千兆字节发展。另一方面，将运算器、控制器等部件集成在一个很小的集成电路上，从而出现了微处理器，进而生产出了微型计算机、笔记本电脑和掌上型等超微型计算机，并使计算机很快进入到寻常百姓家。

 四代计算机的发展速度可以说是突飞猛进的，从电子管到晶体管，从分立元件到集成电路以至微处理器，短短几十年便对人们的生活、科研乃至思维方式产生了不可想象的影响，这离不开计算机发展过程中两位先驱人物的指引：

 图灵是现代计算机逻辑的奠基者，被称为计算机科学之父，人工智能之父，他发明了一种抽象的数学模型–图灵机，只要为其编写好程序，他就可以承担相应的工作，这为后来通用计算机的研制提供了理论基础，也是现代计算机的原型。

 冯诺依曼被称为计算机之父，他提出了冯诺依曼结构，中心思想即存储程序方法，就是设计一个包括存储部件和处理部件的机器,程序存储在存储部件中,处理部件按照存储的程序有条不紊地执行，如今的通用计算机大都采用的是冯诺依曼结构。

### 冯诺依曼结构在四代计算机的发展中占据着主导地位，而随着科学技术的快速发展，冯诺依曼结构计算机的运算速度的提升已达到瓶颈，人们又转而研究非冯诺依曼结构计算机，中心思想是采用并行计算的方法。其中哈弗结构是应用于微处理器的一种非冯诺依曼结构，冯诺依曼结构和哈弗结构的差别：
 冯诺依曼结构是一种将程序指令存储器和数据存储器合并在一起的存储器结构，采用0-1符号编码方法和存储程序方法。这种处理器包括:存储器、 控制器、 运算器、 输入和输出设备几个部分，如今的计算机工作的基本原理大都是冯·诺依曼计算机模式。

  哈弗结构是一种将程序指令存储和数据存储分开的存储器结构。它将程序和数据存储在不同的存储空间中，即程序存储器和数据存储器是两个独立的存储器，每个存储器独立编址、独立访问。用与之对应的至少四条总线对程序和数据进行访问，哈弗结构的数据吞吐量比冯诺依曼结构高一倍，TI公司的生产dsp，51单片机等处理器即采用的哈弗结构。

###根据需求的不同，计算机的种类也产生了多元化的发展：
计算机按设计目可分为：

  • ;通用计算机，是为解决各类问题而设计的计算机，可以进行科学计算、工程计算、数据处理和工业控制等。他是一种用途广泛、结构复杂的计算机；
  • 专用计算机，是为某种特定目的而设计的计算机。例如用于数控机床、轧钢控制、银行存款等的计算机，如ATM机。专用计算机针对性强、效率高、结构比通用计算机简单。
    计算机按规模和性能可分为：
  • ;巨型计算机，规模大、速度快、价格高，一般用于国防和尖端科学、大型科学与工程计算等领域。
  • 大型主机，包括大型机和中型机，价格比较贵，运算速度没有巨型机那样快，一般用于大中型企事业单位。
  • 小型计算机，主要用于一般科学计算、事务处理等，例如某学校可以用一台小型机作为主机，配以数十上百终端机，以满足学生日常学习需要。
  • 微型计算机，体积较小、价廉、易用，典型的就是个人计算机，又称为PC机。
  • 工作站，是介于PC机和小型计算机之间的一种高档微型机，目前已成长为专门处理某类特殊事务的一种独立的计算机系统。
  • 服务器，是一种在网络上为其他计算机提供软件和其他资源的计算机，网络上每一台计算机都可以作为服务器使用。
  • 网络计算机，是一种在网络环境下使用的终端设备，容量大、通信功能强，需要的很多软件都存储在服务器上，因此易于维护。
  • 嵌入式计算机，用于执行特定的功能，通常嵌入在单个微处理器芯片上，程序固化在ROM中，大多微处理器都是嵌入在家电设备中。

### 如今计算机可以说已经发展到了相当成熟的阶段，并且仍然处在高速发展中，同时向着巨型化、微型化、网络化、智能化、新型化等方向发展，正如数十年前，人们难以预见计算机在人类文明的发展过程能够产生的影响，我们亦无法预知在未来的什么时候，计算机会经历怎么样的革命性变化，但是可以肯定的是计算机将继续在人类文明发展史上占据重要的位置，并造福着全人类。
